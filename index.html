Core Systems

KEY FEATURES

Every subsystem is a self-contained ScriptableObject — swap tyres, engines, and suspension setups without touching a line of code.

01

Pacejka Magic Formula Tyre Model

Full Pacejka '94 implementation for both longitudinal and lateral slip forces, with B, C, D, E parameters tunable per tyre compound. Includes friction ellipse combined-slip limiting and slip-circle cosine weighting.

TyreFriction.cs
02

Scriptable Object Tuning Pipeline

Every tunable parameter lives in a dedicated ScriptableObject asset — engine performance, gearing, suspension, brakes, aerodynamics, input sensitivity. Multiple vehicle variants share and override the same assets.

All *Tuning.cs
03

Suspension with Separate Bump / Rebound

Raycast-based spring-damper suspension with independent bump and rebound damper coefficients, preload compression, ride height targeting, and projected-plane normal force calculation for accurate load transfer on slopes.

WheelSuspension.cs
04

Automatic & Manual Transmission

Supports both manual and automatic gearboxes. Auto mode uses RPM-based upshift and a downshift target RPM calculation per gear. Includes clutch engagement simulation with configurable slip delay and gear-change neutral state.

CarEngine.cs
05

RWD / FWD / AWD / LSD Drivetrains

Four drivetrain configurations including a simulated Limited Slip Differential that promotes all-wheel drive when rear slip exceeds a threshold. AWD torque split is configurable with a front axle differential percentage bias.

DrivetrainTuning.cs
06

Ackermann Steering Geometry

Per-wheel steering angles calculated from wheelbase and track width geometry. Blends between parallel and full Ackermann via a configurable percentage. Speed-sensitive steering reduction, countersteer assist, and understeer angle damping built in.

Steering.cs
07

Anti-Roll Bar System

Front and rear anti-roll bars with independent stiffness and damping. Supports equal, compression-weighted, and inverted-weight force distribution modes. Forces applied at hub positions to accurately model body roll stiffness.

AntiRoll.cs
08

Aerodynamic Drag & Downforce

Velocity-squared drag in both longitudinal and lateral axes, plus downforce — each with independent area and coefficient parameters. Forces are direction-aware and applied every fixed timestep for realistic high-speed handling.

VehicleDragTuning.cs
09

Layered Audio Engine

Four-layer audio system — rev up, rev down, redline, and idle — with RPM-driven pitch and volume interpolation. Smooth crossfade logic based on throttle position prevents abrupt transitions at rpm boundaries.

EngineAudioTuning.cs
Why This Framework

BENEFITS

Designed so vehicle feel can be iterated entirely through data assets, with no code changes required between prototyping and final tuning.

01

Zero-Code Vehicle Variants

Every parameter — from final drive ratio to anti-roll stiffness — lives in a ScriptableObject. Creating a new vehicle variant means duplicating and adjusting assets, not touching MonoBehaviour code.

02

Real Tyre Physics Foundation

The Pacejka Magic Formula is the same mathematical model used by real motorsport simulation software. Forces behave correctly under combined slip, load variation, and surface coefficient changes — emergent behaviour follows naturally.

03

Impulse / Average Force Modes

A single toggle switches the entire force pipeline between impulse (instantaneous, exact) and continuous (blended over time) application modes. Useful for switching between tight simulation accuracy and smoother gameplay feel.

04

Modular System Boundaries

Friction, suspension, drivetrain, audio, and effects are cleanly separated. Replacing the tyre model, adding a new drivetrain mode, or swapping the audio system requires changes in exactly one place.

Developer Notes

DEV TIPS

Practical guidance for extending, tuning, and debugging the framework — particularly around the physics pipeline's ordering and data dependencies.

Tuning

Start Pacejka D at 1.0, Tune B First

The B (shape) parameter controls how quickly the tyre reaches peak force. Low B (6–8) = gradual progressive grip. High B (12–16) = sharp spike then drop-off. Set D last as it simply scales the peak — adjust the curve shape first.

// TyreFrictionTuning — suggested starting values linearShapeB = 10f; // grip buildup rate linearStiffnessC = 1.6f; // curve shape (1.4–1.8) linearPeakD = 1.0f; // peak friction coeff linearAsymmetryE = 0.97f; // post-peak drop-off
Physics

Centre of Mass Placement Is Critical

The CoM Transform drives both tyre load distribution and front/rear wheel assignment. Place it low and slightly rearward of geometric centre for realistic weight transfer. Too high causes excessive body roll that no anti-roll stiffness can fully compensate.

// CarController.Start — CoM assigned here if (centreOfMass != null) RB.centerOfMass = centreOfMass.localPosition; // WheelData.AssignWheelPosition() reads this — // must run AFTER CarController.Start()
Tuning

Suspension SpringStiffness vs Vehicle Mass

A sensible starting stiffness is roughly mass × gravity / 4 / rideHeight. At 1500 kg that gives ~9200 N/m per corner at 0.4 m ride height. The default 25000 is stiff — fine for a sports car, but will cause bounce on a heavier vehicle.

// Rough ride frequency target: 1–2 Hz (road car) // f = sqrt(k / m_corner) / (2 * PI) springStiffness = 25000f; // per-corner spring bumpDamperCoeff = 4500f; // ~60–70% critical
Workflow

Use Animation Curves for Exotic Compounds

Toggle useAnimationCurves on TyreFrictionTuning to replace the Pacejka formula entirely with an authored curve. This is useful for highly specific compounds (slicks, rain tyres) where you want a precise visual grip vs. slip shape without fitting Pacejka parameters.

// TyreFrictionTuning public bool useAnimationCurves = true; public AnimationCurve linearSlipCurve; public AnimationCurve lateralSlipCurve; // Curve X = slip ratio (0–1), Y = friction coeff
Tuning

Anti-Roll Stiffness Shapes Handling Balance

Increasing front anti-roll stiffness relative to rear pushes the balance toward understeer. Use the ratio between front and rear, not the absolute values, to set balance. Start both equal then bias from there.

// AntiRollTuning — balance example frontAntiRollStiffness = 12000f; // stiffer front = understeer rearAntiRollStiffness = 8000f; // softer rear = stable // Equal values = neutral baseline
Tuning

countersteerAngleWeight Tunes Oversteer Feel

This parameter in SteeringData scales how aggressively the steering angle opens up when rear slip is detected. At 0 the car has no assisted countersteer. At 3–4 it becomes very arcade. 1.5–2.0 is a good simulation-oriented range that rewards correct technique without being punishing.

Tuning

Final Drive Ratio Controls In-Gear Acceleration

The finalDriveRatio multiplies every gear ratio, re-scaling the entire power delivery without altering the spread between gears. Set the gear spread first, then use final drive to dial the overall torque feel.

// GearingTuning finalDriveRatio = 3.5f; // more torque, lower top speed finalDriveRatio = 2.8f; // less torque, higher top speed // Tune gear spread first, then finalDrive for overall feel
Roadmap

SUGGESTED IMPROVEMENTS

High-value fixes and new features, categorised by impact and implementation complexity.

New Feature

Tyre Temperature Model

Track heat per tyre based on slip work (force x speed difference). Scale the Pacejka D parameter up as tyres warm into their operating window, then degrade beyond peak temperature. Adds a warm-up / graining dynamic relevant to lap simulation.

Impact: HighEffort: Medium
New Feature

Tyre Wear Degradation

Accumulate tyre wear as a scalar that reduces linearPeakD and lateralPeakD over the session. Expose wear rate as a tuning parameter. Opens up compound strategy and long-run pace modelling.

Impact: HighEffort: Medium
New Feature

Weight Transfer History Buffer

Sample and store per-wheel normal force over a rolling window of frames. Use this to drive a chassis pitch / roll animation system decoupled from the physics body, giving smoother visual responses without affecting simulation forces.

Impact: MediumEffort: Medium
New Feature

Transmission: Torque Converter

The current automatic mode engages the clutch directly. A torque converter model (stall torque ratio, lockup speed) would improve low-speed automatic behaviour and creep, particularly relevant for muscle car or SUV feel profiles.

Impact: MediumEffort: High
New Feature

Fuel Load Simulation

Model a fuel tank as a point mass that depletes over time based on throttle position and engine load. The shifting CoM and reducing vehicle mass naturally alter handling balance across a stint without any additional tuning parameters.

Impact: MediumEffort: Low
New Feature

Brake Temperature & Fade

Accumulate brake disc temperature from sustained braking events. Reduce effective brakeTorque as temperature climbs past a threshold, recovering as the discs cool. Pairs naturally with the tyre temperature system for a complete thermal model.

Impact: HighEffort: Medium
Enhancement

Multi-Surface Layer Detection

Extend the raycast to read a texture or tag from the hit surface and automatically set groundSurfaceCoefficient per wheel per frame. Would allow seamless transitions across tarmac, gravel, and kerbs mid-corner.

Impact: HighEffort: Medium
Enhancement

Per-Axle Brake Bias Adjustment

Expose brake balance as a runtime-adjustable value (mapped to a key or axis) rather than a fixed ScriptableObject field. Allows the driver to dial out locking under braking mid-session, mirroring a real brake balance bar.

Impact: MediumEffort: Low
Enhancement

In-Editor Live Telemetry Window

A custom EditorWindow that reads from the active CarController in Play Mode and displays per-wheel slip, normal force, tyre load, and RPM in real time — replacing the OnDrawGizmos sphere readout with a proper dashboard.

Impact: MediumEffort: Medium
Enhancement

Suspension Geometry: Camber & Toe

Add static camber and toe angle offsets to each wheel's tuning data. Factor these into the slip angle calculation so corner-entry and mid-corner lateral force balance can be tuned geometrically as well as through spring rates.

Impact: HighEffort: Medium
Enhancement

Force Feedback Output API

Expose a lightweight data struct each fixed frame containing steering rack force, wheel vibration magnitude, and kerb impact transients. Any force-feedback peripheral or haptic system can consume this without knowing anything about the internals.

Impact: MediumEffort: Low
Enhancement

Replay & Data Logging System

Record per-frame telemetry (speed, RPM, slip ratios, G-forces, gear) to a lightweight buffer during a run. Play back the ghost as a visual overlay and export the CSV for post-session analysis — useful for tuning iteration and lap comparison.

Impact: HighEffort: High

```
Enhancement

Tyre Relaxation Length Filter

Lateral force currently reaches steady-state instantaneously regardless of vehicle speed. In reality, tyres build lateral force over a physical travel distance — the relaxation length, typically 0.3–0.5 m for road car tyres. This produces a speed-dependent time constant: τ = λ / |V|. At low speed the tyre responds sluggishly (correct — parking lot cornering feels heavy); at high speed it converges almost instantly. Implement as a first-order lag on filteredLateralForce in CalculateLateralSlipForce, driven by |V| / relaxationLength per fixed timestep. Also naturally smooths the lateral discontinuity that currently occurs at the staticFrictionThreshold speed boundary. The Pacejka formula itself requires no changes.

Impact: High Effort: Low TyreFriction.cs TyreFrictionTuning.cs WheelData.cs
```
